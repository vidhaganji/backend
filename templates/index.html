Fitness Tracker Backend

JWT Token Authetication

  1. Purpose: Our fitness tracker contains private information of our users. To prevent non-admin, other users, and anyone else without credentials from acessing their data, we used JWT Authentication.
  2. To learn about how the specific anatomy of our system look at its diagram above.
  3. Individual user vs Admin roles - Individual users login to access their own personal data and CRUD functions for their own accounts. Admin users login to access the data and CRUD functions for all users. this allows admins to monitor the site without compromising user information to normal users thanks to the JWT Token.
  4. JWT Storage: Our JWTs are stored in the form of cookies instead of local storage. That way third-party scripts (ex: Github social sharing buttons) cannot access the token and protects against XSS (Cross-site scripting) attacks.
  5. API and Microservice using Flask. Mastering development techniques in creating and consuming RESTful APIs
